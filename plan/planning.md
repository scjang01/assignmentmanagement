

**앱 개발 기획서**

**과제 서포트 앱**

*AssignmentHelper*

개발 환경: Android Studio  |  참여 인원: 10명  |  작성자: 황다현

# **1\. 기획 동기**

대학교 학사 생활에서 과제 관리는 빠질 수 없는 중요한 업무임에도 불구하고, 현재 사이버캠퍼스 및 코스모스 앱은 과제에 특화된 접근성을 제공하지 않아 다음과 같은 불편함이 존재한다.

* 각 과제를 확인하기 위해 사이버캠퍼스 웹 또는 코스모스 앱에 직접 접속하여 해당 강의 → 해당 주차 → 해당 과제 순으로 탐색해야 하는 번거로움이 존재한다.  
* 접근성 저하로 인해 마감 직전이나 기한 초과 후에야 과제를 인식하는 상황이 빈번하게 발생한다.  
* 조별과제 시 이름 · 학번 · 과 · 연락처 외에도 역량, 가용 시간 등 사전 조율 사항이 많아 초기 진입 장벽이 높다.

본 기획서는 이러한 문제를 해결하기 위해 개인과제·조별과제·녹화강의 수강 과제를 포괄적으로 지원하는 전용 앱의 개발을 목적으로 작성되었다.

# **4\. 기능 구성 상세**

## **4.1  모의 학사행정시스템 (사이버캠퍼스 서버)**

기존 사이버캠퍼스를 모의로 별도 구축하여 아래 데이터를 앱과 자동 연동한다. DB를 먼저 설계하고 이후 웹 인터페이스를 구축하는 순서로 진행한다.

* 개인 과제 등록 및 마감일 관리  
* 녹화강의 업로드 및 수강 데드라인 관리  
* 강의 공지 등록 (유형: 긴급 / 시험 / 휴 · 보강 / 기타)  
* 자동 연동: 신규 과제·공지 등록 시 앱에 푸시 알림 발송

## **4.2  앱 서버**

클라이언트(앱)는 입력과 결과 조회만 담당하며, 주요 연산과 데이터 저장은 앱 서버가 처리한다. 개인 핀 등 일부 사용자 맞춤 데이터만 로컬에 저장한다.

## **4.3  메인 기능 \#1 — 과제 리마인더**

핵심 기능. 모의 사이버캠퍼스와 자동 연동되어 과제 정보를 불러오며, 알림 주기는 사전 설정·수동 편집이 가능하다.

* 연동 데이터: 과목명 / 과제명 / 마감일 / 사이버캠퍼스 바로가기 링크  
* 리마인드 알림: 미완료 상태일 때 설정된 시기·주기에 따라 푸시 알림 발송  
* 기한 초과 과제: 기존 주기 무시, 하루 2회 확인 알림 자동 발송  
* 녹화강의 수강 과제: 현재 수강 진행도 표시 및 리마인드 알림 지원  
* 각 과제 항목: 편집 / 완료 처리 / 삭제 처리 버튼 제공

## **4.4  메인 기능 \#2 — 프리타임 캘린더**

각 사용자가 자신의 자유 시간(프리타임)을 직접 등록해두는 개인 캘린더.   
구글 캘린더 연동은 조사상 낮은 사용 빈도 및 개인정보 노출 우려로 채택하지 않는다.

* 월\~일 7요일 × 24시간, 1시간 단위로 구성  
* 클릭 또는 드래그로 간편하게 프리타임 설정  
* 공개 / 비공개 설정 가능 — 비공개 시 그룹 캘린더에서 해당 사용자 제외

## **4.5  메인 기능 \#3 — 그룹 생성 / 관리**

조별과제를 위한 그룹을 생성하고 구성원을 관리한다.

* 그룹 생성 시 타 사용자의 ID를 검색하여 구성원 추가  
* 그룹 생성 이후 구성원 추가 초대 / 퇴장 처리 가능  
* 그룹 생성자 또는 관리자만 자신 외 구성원 퇴장 가능 (설정으로 제어)  
* 과제 완료 등 목적 달성 시 그룹 삭제 가능

## **4.6  메인 기능 \#4 — 그룹 캘린더 / 공용 프리타임 / 투표**

그룹 내 구성원의 프리타임 교집합을 산출하여 미팅 가능 시간을 추천하고, 특정 시간대에 대한 투표·확정 기능을 제공한다.

* 구성원 전원의 공개 프리타임을 비트마스크 방식으로 처리하여 교집합 산출  
* 공용 프리타임 생성 시 해당 시간대를 전체 구성원에게 알림으로 발송  
* 구성원은 추천된 시간대에 '확인 반응'(투표)을 보낼 수 있음  
* 투표 결과를 기반으로 특정 시간대를 미팅 타임으로 확정  
* 수동 수정 버튼을 통해 공용 프리타임 편집 가능 — 수정 후 재등록 시 투표 프로세스 재시작  
* 캘린더 이미지 저장 기능으로 외부 공유 가능

## **4.7  메인 기능 \#5 — 그룹 게시판**

그룹 내 할 일·일정·파일·링크를 한 곳에서 관리하는 공유 게시판.

* 조별 할 일(TODOLIST) 또는 마일스톤으로 사용 가능한 과제 추가  
* 과제 추가 시 알림 수신 대상(역할 또는 개인)을 지정 가능  
* '팔로우' 기능: 지정된 멤버는 기본 팔로우 상태, 미지정 멤버도 직접 팔로우하여 알림 수신 가능  
* 편집 / 완료 처리 / 삭제: 그룹 생성자·관리자·해당 역할 사용자만 수행 가능  
* 파일·이미지 업로드: 카카오톡 파일 유효기간 문제를 보완하는 공용 파일 저장소 역할  
* 바로가기 링크 등록: 깃허브 리포지토리·웹페이지·유튜브 등 공용 링크 모음

## **4.8  공지 알림 기능**

모의 사이버캠퍼스에 공지가 등록될 때 앱 알림 발송 및 미열람 공지 리마인드 기능을 제공한다.

* 공지 유형: 긴급 / 시험 / 휴·보강 / 기타 (네이버카페 말머리 방식)  
* 메인 화면 상단에 '새 공지가 n건 있습니다.' 배너 표시  
* 미열람 공지 존재 시 리마인드 알림 추가 발송  
* 공지 목록: 미열람 / 전체 탭 분리, 과목별 필터링 및 정렬 지원

## **4.9  보조 기능 — 개인 계정 로그인**

* Firebase Authentication을 이용한 로그인 시스템  
* 로그인 이력 존재 시 자동 로그인 처리  
* 앱 내 표시 정보: 아이디, 이름, 학번

# **5\. 화면 구성 (UI 흐름)**

| 화면 | 주요 구성 요소 |
| :---- | :---- |
| 스플래시 화면 | 앱 실행 시 로딩 화면 → 로그인 화면으로 자동 전환 |
| 로그인 화면 | Firebase 기반 로그인, 자동 로그인 지원 |
| 메인 화면 | 상단: 공지 알림 배너 / 중단: 과제 목록 리스트뷰 / 하단: 네비게이션 바(과제 · 조별과제 · 마이페이지) |
| 과제 세부 설정 화면 | 과제 정보(사이버캠퍼스 연동), 바로가기 링크, 리마인드 알림 설정 (스마트폰 알람 UI와 유사) |
| 녹화강의 세부 화면 | 수강 진행도, 바로가기 링크, 리마인드 설정 |
| 공지 목록 화면 | 미열람 공지 최우선 표기, 과목별 필터링, 미열람/전체 탭 분리 |
| 마이페이지 화면 | 아이디 · 이름 · 학번, 프리타임 설정 버튼, 알림 설정(타입 · 주기 기본값 변경) |
| 프리타임 설정 화면 | 7요일 × 24시간 격자, 클릭/드래그 선택, 초기화·저장 버튼 |
| 조별과제 목록 화면 | 그룹 리스트, 그룹 추가(+) 버튼, 구성원 ID 검색 및 추가 |
| 조별과제 세부 화면 | 상단 탭: 게시판 / 공용 프리타임 / 투표 결과 |
| 그룹 게시판 화면 | 과제/할일 목록, \+ 버튼으로 추가, 파일 · 이미지 · 링크 업로드, 개인 핀 · 정렬 |
| 공용 프리타임 화면 | 공용 프리타임 생성 버튼, 추천 시간대 표시, 확인 반응(투표), 수동 수정 |

# **6\. 개발 계획 및 단계별 구조**

기능 구현은 아래 단계 순서로 점진적으로 확장하며, 어느 단계에서 개발이 종료되더라도 해당 단계까지의 기능은 완성본으로 인정한다.

| 단계 | 내용 | 비고 |
| :---- | :---- | :---- |
| 1단계 | 개인 과제 메모 / 리마인더 구현 \+ 모의 사이버캠퍼스 자동 연동 | 핵심 기능 MVP |
| 2단계 | 공지 알림 기능 \+ 녹화강의 수강 과제 연동 | 모의 서버 기능 확장 |
| 3단계 | 프리타임 캘린더 \+ 그룹 생성·관리 \+ 공용 프리타임 (교집합 산출) | 조별 기능 기반 |
| 4단계 | 공용 프리타임 시간대 투표·확정 기능 | 조별 일정 조율 완성 |
| 5단계 | 그룹 게시판 구현 (과제 추가, 팔로우, 알림) | 조별 협업 기능 |
| 6단계 | 그룹 게시판 확장 (파일 · 이미지 · 바로가기 링크 첨부) | 추가 편의 기능 |
| (선택) 7단계 | 앱 내 채팅 시스템 — 일정 및 여유에 따라 추가 개발 검토 | 보류 |

* 기기 확장: 데이터가 서버 DB에 저장되므로 데스크탑·태블릿 등 다양한 기기로 연동 확장 가능
